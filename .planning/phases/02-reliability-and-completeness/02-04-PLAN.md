Audit and fix the following plot functions for edge cases:

1. **`plot_aop_lifetime()`** (returns `tuple[str, str, str]` — created histogram, modified histogram, scatter):
   - Add try/except wrapper around the entire function body with `create_fallback_plot()` for each of the 3 sub-plots on error
   - Handle empty DataFrame from SPARQL query (check `if df.empty:` and return 3 fallback plots)
   - Handle `pd.to_datetime()` failures on malformed date strings by wrapping in try/except with `errors='coerce'`
   - Ensure each sub-plot's DataFrame is also cached in `_plot_data_cache` (added in Plan 02-02)
   - If any individual sub-plot fails, return a fallback for that sub-plot while allowing others to succeed

2. **`plot_ke_components()`**:
   - Add error handling for empty SPARQL results (empty DataFrame)
   - Wrap in try/except at the function level
   - If the query returns no data, return 2 fallback plots (absolute + delta)

3. **`plot_ke_components_percentage()`**:
   - Same treatment as ke_components — handle empty results
   - Division-by-zero guard: if total KE count is 0 for a version, skip that version rather than dividing by zero

4. **General audit** — scan ALL trend plot functions for these patterns:
   - Missing `try/except` wrappers → add them
   - `pd.to_datetime()` calls without `errors='coerce'` → add the parameter
   - DataFrame operations that assume non-empty data → add `if df.empty:` guards
   - Division operations without zero-guards → add guards

**Pattern for error handling** (from existing reliable functions):
```python
def plot_some_function():
    try:
        # ... computation ...
        return result
    except Exception as e:
        logger.error(f"Failed to generate some_function plot: {str(e)}")
        return create_fallback_plot("Plot Title", str(e))
```

For functions returning tuples, return a tuple of fallback plots matching the expected return shape.
