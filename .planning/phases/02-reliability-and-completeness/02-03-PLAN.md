   ```json
   {
     "plot_key": {
       "title": "Plot Title",
       "description": "What this measures — plain language, ~1 paragraph.",
       "data_source": "How data is sourced — SPARQL endpoint, query pattern, processing steps.",
       "limitations": "Known limitations — honest disclosure for researchers.",
       "sparql": "Representative SPARQL query text (static template with {placeholders} where dynamic)."
     }
   }
   ```

   **Latest page plots to include** (~11 entries):
   - `latest_entity_counts`: Counts instances of each entity class (AOP, KE, KER, Stressor) and unique authors
   - `latest_aop_connectivity`: AOP connectivity via shared Key Events (connected vs isolated)
   - `latest_avg_per_aop`: Average KEs and KERs per AOP
   - `latest_ke_components`: Distribution of Process/Object/Action annotations on KEs
   - `latest_ke_annotation_depth`: How many KE Component annotations each KE has (0, 1, 2, or 3)
   - `latest_aop_completeness`: Property completeness distribution for AOPs
   - `latest_aop_completeness_by_status`: AOP completeness grouped by OECD approval status
   - `latest_ke_completeness_by_status`: KE completeness grouped by parent AOP's OECD status
   - `latest_ker_completeness_by_status`: KER completeness grouped by parent AOP's OECD status
   - `latest_process_usage`: Biological process ontology term usage in KE annotations
   - `latest_object_usage`: Biological object ontology term usage in KE annotations

   **Trends page plots to include** (~15 entries):
   - `aop_entity_counts`: Entity count evolution over time (AOPs, KEs, KERs, Stressors)
   - `aop_property_presence`: AOP property presence tracking over time
   - `ke_property_presence`: KE property presence tracking over time
   - `ker_property_presence`: KER property presence tracking over time
   - `stressor_property_presence`: Stressor property presence tracking over time
   - `entity_completeness_trends`: Average completeness percentage per entity type over time
   - `aop_completeness_boxplot`: Distribution of composite AOP completeness scores per version
   - `oecd_completeness_trend`: Mean completeness per OECD status across versions
   - `average_components_per_aop`: Average KEs and KERs per AOP over time
   - `aop_network_density`: KE-KER graph density ratio over time
   - `aop_authors`: Unique AOP author count over time
   - `aop_lifetime`: AOP creation and modification temporal patterns
   - `ke_component_annotations`: Process/Object/Action annotation counts over time
   - `bio_annotations`: Biological process and object annotation trends
   - `kes_by_kec_count`: Distribution of KEs by number of KE Components over time

   **For SPARQL queries:** Use representative static versions. For queries that are dynamically generated from property_labels.csv, show the template with `{property_list}` or `{version_graph}` placeholders and note in limitations that the actual query varies.

   **For limitations, always include relevant items like:**
   - Data reflects RDF transformation, not raw AOP-Wiki database
   - Author counts are unique creator URIs, may not match AOP-Wiki user accounts
   - Completeness excludes 100%-present properties (always filled in)
   - OECD status data is version-specific and may change between releases
   - Network density depends on explicitly declared KE-KER relationships

2. Create `templates/macros/methodology.html` with a Jinja2 macro:
   ```html
   {% macro methodology_note(notes, plot_key) %}
   {% if plot_key in notes %}
   {% set note = notes[plot_key] %}
   <details class="methodology-note">
       <summary>Methodology</summary>
       <div class="methodology-content">
           <p><strong>What this measures:</strong> {{ note.description }}</p>
           <p><strong>Data source:</strong> {{ note.data_source }}</p>
           <p><strong>Known limitations:</strong> {{ note.limitations }}</p>
           {% if note.sparql %}
           <details class="sparql-query">
               <summary>View SPARQL Query</summary>
               <pre><code>{{ note.sparql }}</code></pre>
           </details>
           {% endif %}
       </div>
   </details>
   {% endif %}
   {% endmacro %}
   ```
