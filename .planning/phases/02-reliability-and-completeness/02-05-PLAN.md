<task type="auto">
  <name>Task 2: Split boxplot query to avoid Virtuoso MaxResultRows truncation</name>
  <files>plots/trends_plots.py</files>
  <action>
The `plot_aop_completeness_boxplot()` function (~line 2340) uses two combined UNION queries (`query_combined_presence` and `query_combined_totals`) that span ALL graphs and ALL entity types (AOP, KE, KER) in a single request. When the total result set exceeds Virtuoso's MaxResultRows, results are silently truncated — newer versions (post-2020) are dropped.

**Apply the same per-version parallel query pattern used in `plot_oecd_completeness_trend()` (02-01 established pattern):**

1. Get all versions via `get_all_versions()`.
2. For each version, run the presence and totals queries scoped to that single graph:
   - Replace `GRAPH ?graph { ... } FILTER(STRSTARTS(...))` with `GRAPH <{specific_graph_uri}> { ... }`
   - This dramatically reduces result set size per query (one version's data instead of all).
3. Use `ThreadPoolExecutor(max_workers=4)` for parallel execution across versions (same pattern as OECD trend).
4. Merge per-version results into the combined DataFrame for boxplot generation.

**Specific changes to `plot_aop_completeness_boxplot()`:**

a) Extract the inner query logic into a helper function `_query_boxplot_version(version, aop_props, ke_props, ker_props, all_props_filter)` that:
   - Takes a single version (graph URI)
   - Runs the presence query scoped to that graph
   - Runs the totals query scoped to that graph
   - Returns the per-version results (or empty DataFrame on error)

b) Replace the two `run_sparql_query()` calls on the combined queries with:
```python
versions = get_all_versions()
with ThreadPoolExecutor(max_workers=4) as executor:
    futures = {
        executor.submit(_query_boxplot_version, v, ...): v
        for v in versions
    }
    # Collect results
```

c) Import `ThreadPoolExecutor` from `concurrent.futures` if not already imported.

d) Keep the existing boxplot generation code (everything after the query results are assembled). Only the data-fetching part changes.

e) Add logging: `logger.info(f"Boxplot: querying {len(versions)} versions in parallel")` and per-version error logging.

f) Update the `_plot_data_cache` key to include all versions' data.

**Do NOT change the boxplot visualization logic** — only the SPARQL query execution strategy.
