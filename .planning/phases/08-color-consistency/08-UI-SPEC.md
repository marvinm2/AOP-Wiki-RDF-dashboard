### Plot Color Contract (Phase 8 scope)

This is the core deliverable of Phase 8. Every plot must conform to one of these treatments:

| Treatment | Color Value | When Applied | Plotly Parameter |
|-----------|-------------|--------------|------------------|
| Single-metric | `BRAND_COLORS['blue']` (#307BBF) | One series, one count, one average, one density | `fig.update_traces(marker_color=BRAND_COLORS['blue'])` for bars; `color_discrete_sequence=[BRAND_COLORS['blue']]` for lines |
| Categorical | `BRAND_COLORS['palette']` (10-color array) | Distinct named groups: entity types, ontology sources, components | `color_discrete_sequence=BRAND_COLORS['palette']` |
| Semantic | `BRAND_COLORS['type_colors']` or `BRAND_COLORS['oecd_status']` | Biologically or status-meaningful categories | `color_discrete_map=BRAND_COLORS['type_colors']` |

### Documentation Copy (COLOR-02)

The Color Decision Framework to be added to `.claude/colors.md` uses this exact decision tree text:

1. **Is the data categorical?** (distinct named groups: entity types, ontology sources, OECD statuses)
   - YES: Use `BRAND_COLORS['palette']` or semantic mapping (`type_colors`, `oecd_status`)
   - NO: Continue to step 2

2. **Is it a single metric or series?** (one count, one average, one density)
   - YES: Use `BRAND_COLORS['blue']` (#307BBF)
   - NO: Continue to step 3

3. **Are there multiple views of the same entity?** (absolute + delta, created + modified)
   - YES: Use `BRAND_COLORS['blue']` for ALL outputs (consistency)
   - NO: Consult team
