**1. Create `.claude/add-a-plot.md`:**

Create a comprehensive add-a-plot checklist. Per user decisions: checklist format only (no worked example walkthrough), aimed at both human contributors and Claude equally, with precise file paths and patterns but also readable narrative.

The checklist must cover BOTH plot types with separate sections:

**Section A: Adding a Latest Data Plot (Database Snapshot)**

Checklist items with exact file paths, function signatures, and code patterns:

1. **Implement the plot function** in `plots/latest_plots.py`
   - Function signature: `def plot_latest_<name>(version: str = None) -> str:`
   - Use `_build_graph_filter(version)` for SPARQL WHERE/ORDER generation
   - Cache DataFrame: `_plot_data_cache[f'latest_<name>_{version or "latest"}'] = df`
   - Return: `pio.to_html(fig, full_html=False, include_plotlyjs='cdn')`
   - Brand colors: import `BRAND_COLORS` from `plots.shared`

2. **Export from package** in `plots/__init__.py`
   - Add to the `from .latest_plots import (...)` block
   - Add to the `__all__` list

3. **Add lazy-loading API route** in `app.py`
   - Add entry to the `plot_map` dictionary in the `/api/plot/<plot_name>` endpoint (around line 1250+)
   - Key: `'latest_<name>'`, Value: `lambda: plot_latest_<name>(request.args.get('version'))`

4. **Add CSV download route** in `app.py`
   - Pattern: `@app.route("/download/latest_<name>")`
   - Use `get_csv_with_metadata('latest_<name>', include_metadata=True)`

5. **Add HTML container** in `templates/latest.html`
   - Add a `<div class="lazy-plot" data-plot-name="latest_<name>">` with skeleton loader
   - Include download button linking to `/download/latest_<name>`

6. **Register for version selector** in `static/js/version-selector.js`
   - Add `'latest_<name>'` to the `versionedPlots` array

**Section B: Adding a Historical Trends Plot**

Checklist items:

1. **Implement the plot function** in `plots/trends_plots.py`
   - For diff plots: return `(absolute_html, delta_html, data_df)` — 3-tuple
   - For property presence: return `(absolute_html, percentage_html)` — 2-tuple
   - Cache DataFrames: `_plot_data_cache['<name>_absolute'] = df` and `_plot_data_cache['<name>_delta'] = df`

2. **Export from package** in `plots/__init__.py`
   - Add to the `from .trends_plots import (...)` block
   - Add to `__all__`

3. **Add to startup computation** in `app.py`
   - Add task tuple to `plot_tasks` list in `compute_plots_parallel()` (around line 164+)
   - Pattern: `('plot_name', plot_function_name)`

4. **Unpack results** in `app.py` (around line 230+)
   - For 3-tuple: `graph_<name>_abs, graph_<name>_delta, _ = plot_results.get('<name>', ("", "", None))`
   - For 2-tuple: `graph_<name>_abs, graph_<name>_pct = plot_results.get('<name>', ("", ""))`

5. **Add to plot_map** in `app.py` `/api/plot/<plot_name>` endpoint
   - Add entries for both absolute and delta/percentage variants

6. **Add download routes** in `app.py`
   - Two routes: `/download/trend/<name>_absolute` and `/download/trend/<name>_delta`

7. **Add HTML containers** in `templates/trends_page.html`
   - Side-by-side layout with two `lazy-plot` divs
   - Include download buttons for both variants

**Section C: Naming Conventions**
- Function: `plot_latest_<name>()` or `plot_<name>()`
- Cache key: `latest_<name>_<version>` or `<name>_absolute` / `<name>_delta`
- Route: `/download/latest_<name>` or `/download/trend/<name>_absolute`
- Template div: `data-plot-name="latest_<name>"` or `data-plot-name="<name>_absolute"`

**Section D: Common Patterns**
- SPARQL query structure and `run_sparql_query_with_retry()` usage
- Error handling: `safe_plot_execution()` wrapper
- Brand colors reference: `BRAND_COLORS` dict in `plots/shared.py` and `.claude/colors.md`
- Plotly layout defaults: transparent background, VHP brand header color for text
