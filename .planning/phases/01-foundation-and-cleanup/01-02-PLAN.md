**1. Implement VersionedPlotCache in plots/shared.py:**

Add a `VersionedPlotCache` class near the top of `plots/shared.py` (after imports, before existing code). This replaces the plain dict caches (`_plot_data_cache = {}` and `_plot_figure_cache = {}`).

Class requirements (from user decisions):
- TTL-based eviction (use 30 minutes / 1800 seconds)
- Hard cap on number of versions cached simultaneously (use 5 historical versions max)
- Latest/current version is pinned and never evicted
- Thread-safe with `threading.Lock` for Gunicorn gthread workers

Use the implementation pattern from 01-RESEARCH.md Pattern 1 (VersionedPlotCache with OrderedDict). Key methods: `get(key, default=None)`, `set(key, value)`, `pin_version(version)`, `__contains__(key)`, `__setitem__(key, value)`, `__getitem__(key)`.

Replace the existing cache dict definitions:
- Change `_plot_data_cache = {}` to `_plot_data_cache = VersionedPlotCache(max_versions=5, ttl_seconds=1800)`
- Change `_plot_figure_cache = {}` to `_plot_figure_cache = VersionedPlotCache(max_versions=5, ttl_seconds=1800)`

IMPORTANT: The VersionedPlotCache must support dict-like `[]` access (via `__getitem__`/`__setitem__`/`__contains__`) for backward compatibility — existing code uses `_plot_data_cache['key'] = value` and `'key' in _plot_data_cache` syntax extensively. Do NOT rewrite all call sites. Make the class a drop-in replacement.

**2. Centralize JSON logging in config.py:**

Add a `configure_logging()` function to `config.py` using `pythonjsonlogger.jsonlogger.JsonFormatter` (already installed as python-json-logger 2.0.7).

Use the pattern from 01-RESEARCH.md Pattern 3:
- Handler: `logging.StreamHandler(sys.stdout)` for Docker log collection
- Format fields: timestamp (ISO 8601), level, logger, message, module, funcName
- Rename fields: asctime->timestamp, levelname->level, name->logger
- Clear existing handlers before adding the new one

Remove the duplicate `logging.basicConfig()` calls:
- Remove `logging.basicConfig(...)` from `app.py` (line ~54 area — verify exact location)
- Remove `logging.basicConfig(...)` from `plots/shared.py` (line ~69 area — verify exact location)
- Add a single call to `configure_logging()` early in `app.py` before any other imports that might log

**3. Create gunicorn.conf.py:**

Create `gunicorn.conf.py` at project root with the configuration from 01-RESEARCH.md Pattern 2:
- `bind = "0.0.0.0:5000"`
- `workers = 2` (containerized deployment, scale by adding containers)
- `worker_class = "gthread"`
- `threads = 4` (8 total concurrent handlers)
- `preload_app = True` (critical: runs compute_plots_parallel() once in master, shared via COW fork)
- `timeout = 120` (AOP completeness takes ~75s, need headroom)
- `graceful_timeout = 30`
- `keepalive = 2`
- `worker_tmp_dir = "/dev/shm"` (prevents Docker tmpfs heartbeat failures)
- `accesslog = "-"` and `errorlog = "-"` (stdout/stderr for Docker)
- `loglevel = "info"`

Remove any `FLASK_RUN_HOST` or `FLASK_RUN_PORT` env vars from the Dockerfile since they are Flask-dev-server-specific and no longer needed.
