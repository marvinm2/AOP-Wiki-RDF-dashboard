  <name>Task 1: Standardize latest_plots.py — remove template overrides and use render_plot_html</name>
  <files>plots/latest_plots.py</files>
  <action>
This file has ~20 plot functions and ~20 `to_html()` / `pio.to_html()` calls. Apply these mechanical transformations:

**Step 1: Add render_plot_html to imports.**
At the top of latest_plots.py, add `render_plot_html` to the import from `plots.shared` (or add a new import line if needed).

**Step 2: Remove all explicit `template="plotly_white"` from `update_layout()` calls.**
Search for `template="plotly_white"` or `template='plotly_white'` in the file and REMOVE that parameter from every `update_layout()` call. The default template (`plotly_white+vhp4safety`) now handles this. If `template=` is the ONLY parameter in an `update_layout()` call, remove the entire call.

**Step 3: Remove redundant `update_layout()` parameters that the template already provides.**
For each `update_layout()` call, remove these parameters if present (they match the template defaults):
- `hovermode="x unified"` (template default)
- `autosize=True` (template default)
- `paper_bgcolor="white"` or `paper_bgcolor="rgba(0,0,0,0)"` (template default is white)
- `plot_bgcolor="white"` or `plot_bgcolor="rgba(0,0,0,0)"` (template default is white)
- `font=dict(...)` if it matches the template font (family="Arial, sans-serif", size=13, color="#29235C")

KEEP these per-plot overrides that are intentionally different from defaults:
- `margin=dict(l=300)` or other custom margins for horizontal bar charts with long labels
- `hovermode="closest"` for bar charts and scatter plots (intentional per-plot override)
- `showlegend=False` for pie charts
- `height=` for plots that need dynamic height
- `barmode=` settings
- `xaxis_title=`, `yaxis_title=` axis labels
- `title=` — REMOVE plot titles since HTML headings provide them (per research discretion recommendation). However, if the title is used in a way that's critical to understanding (e.g., subtitle information), keep it but simplify.

**Step 4: Replace all `pio.to_html()` and `fig.to_html()` calls with `render_plot_html(fig)`.**
Search for patterns:
- `pio.to_html(fig, full_html=False, include_plotlyjs=..., config=...)` → `render_plot_html(fig)`
- `pio.to_html(fig, full_html=False, include_plotlyjs=...)` → `render_plot_html(fig)`
- `fig.to_html(full_html=False, include_plotlyjs=...)` → `render_plot_html(fig)`

The first call on each page that previously used `include_plotlyjs="cdn"` should now also use `render_plot_html(fig)` with the default `include_plotlyjs=False`, since the CDN script tag is loaded in `base.html` head_extra block. However, if there are pages that load plotly via CDN in the template, double-check that the CDN script is present.

**Step 5: Remove Plotly titles that duplicate HTML headings.**
Search for `title=` in `px.bar()`, `px.line()`, `px.scatter()`, `px.pie()`, `px.box()` calls and in `update_layout(title=...)`. Remove the title parameter. The existing `<h3>` headings in the HTML plot containers provide the title.

**Important edge cases:**
- `create_fallback_plot()` in shared.py also calls `to_html()` — update that separately if it exists, or leave it since it's a fallback path.
- The `plot_latest_network_density()` function returns raw gauge figure — check if it uses `to_html()`.
- Some functions return tuples of plots — update both plot renderings.
